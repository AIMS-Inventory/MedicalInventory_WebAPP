# NASA HUNCH Presentation Guide
## Visual Inventory Tracking System - Quick Explanation

### **What This System Does (30 seconds)**
"This is a web-based inventory tracking system for the ISS that uses a camera on a Jetson Nano to detect when medical supplies are removed. It logs usage, tracks inventory, and predicts when supplies will run out."

---

## **Key Features to Demonstrate**

### 1. **Login System** (Simple Security)
**What to say:**
- "We implemented basic authentication to protect the system"
- "The password is hardcoded for prototype demonstration: `67`"
- "In production, this would connect to a database"

**How it works:**
- User enters password → JavaScript checks if it matches → If correct, shows dashboard
- Uses `sessionStorage` to remember login (stays logged in after refresh)

**Code location:** `script.js` lines 54-82

---

### 2. **Dashboard with Video Stream**
**What to say:**
- "This is the main control center"
- "The video placeholder shows where the live Jetson Nano camera feed will appear"
- "Below that, we see the current inventory table"

**How it works:**
- HTML creates the structure (`index.html`)
- CSS styles it to look like a medical/aerospace interface (`style.css`)
- JavaScript loads data from browser storage and displays it

**Code location:** `index.html` lines 50-90

---

### 3. **Inventory Log** (Reads from JSON)
**What to say:**
- "This shows a history of when items were removed"
- "The data comes from `logs.json` which simulates what the Jetson Nano would write to an SD card"
- "Each entry shows: timestamp, item name, quantity removed, which astronaut, and location"

**How it works:**
- JavaScript uses `fetch()` to read `logs.json` file
- Displays data in a table format
- Updates automatically when you refresh

**Code location:** `script.js` lines 280-320

---

### 4. **AI Predictions Panel**
**What to say:**
- "This simulates AI predictions for when supplies will run out"
- "The data comes from `predictions.json` which would be generated by a machine learning model on the Jetson"
- "Cards are color-coded: Green = OK, Yellow = Warning, Red = Critical"

**How it works:**
- Reads `predictions.json` file
- Calculates urgency based on days until depletion
- Displays as cards with color coding

**Code location:** `script.js` lines 330-380

---

### 5. **Smart Scanner (OCR Simulation)**
**What to say:**
- "This simulates optical character recognition"
- "You paste text from a medication label, and it extracts key information using pattern matching"
- "Uses Regular Expressions (Regex) to find dates, dosages, and quantities"

**How it works:**
- User pastes text → JavaScript searches for patterns like "2024-12-15" (dates) or "200mg" (dosage)
- Auto-fills the form fields
- User verifies and saves to inventory

**Code location:** `script.js` lines 121-160

## **Technical Architecture (If Asked)**

### **Frontend (What You See)**
- **HTML** (`index.html`): Structure of the webpage
- **CSS** (`style.css`): Styling - dark theme, high contrast for medical use
- **JavaScript** (`script.js`): Logic - handles login, loads data, displays tables

### **Backend (Optional - Flask)**
- **Python Flask** (`app.py`): Server that could handle API requests
- Currently, the app works **client-side only** (no server needed)
- Flask is included for future integration with Jetson Nano

### **Data Storage**
- **LocalStorage**: Browser storage for inventory (persists after refresh)
- **JSON Files**: `logs.json` and `predictions.json` simulate data from Jetson Nano

## **Common Questions & Answers**

**Q: "How does authentication work?"**
A: "Simple password check in JavaScript. For prototype, password is `67`. In production, we'd use a database."

**Q: "Where does the data come from?"**
A: "Currently from JSON files that simulate what the Jetson Nano would write. The camera system would detect item removal and log it to `logs.json` on the SD card."

**Q: "How does the camera integration work?"**
A: "The video placeholder shows where the stream will appear. When the Jetson Nano is connected, we'll point it to `http://jetson-ip:8080/stream`. The camera status indicator will show ACTIVE/OFFLINE based on connection."

**Q: "What technologies did you use?"**
A: "Vanilla JavaScript, HTML, CSS - no frameworks. This makes it easy to explain and maintain. Python Flask is included for future backend integration."

**Q: "How does the AI prediction work?"**
A: "Currently simulated with hardcoded data in `predictions.json`. In full implementation, a machine learning model on the Jetson would analyze usage patterns and predict depletion dates."

## **If Something Breaks During Demo**

**Login doesn't work?**

- Make sure you're using password: `nasa2024`

**JSON files don't load?**
- Make sure you're running a local server (not opening file directly)
- Use: `npx http-server -p 8000` then go to `http://localhost:8000`

**Styling looks wrong?**
- Hard refresh: Ctrl+Shift+R (clears cache)
